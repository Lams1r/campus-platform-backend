# Campus Platform 自动化测试设计

## 测试目标

本测试套件部署在测试服务器 `43.134.21.21`，用于自动测试前端服务器 `http://8.138.10.124` 和后端 API 服务器 `http://114.132.77.204:8080/api`。它以黑盒测试为主，不登录对方服务器、不读取对方数据库，只通过浏览器同样能访问的 HTTP 页面和公开 API 接口判断系统质量。

## 测试类型

- 冒烟测试：最快验证前端首页、核心页面和基础 API 是否可用。
- 黑盒部署测试：从公网视角检查 Nginx、SPA 刷新、静态资源、API 转发。
- 部署质量测试：检查 Content-Type、CORS 预检、API 错误页、静态资源可访问性。
- 前端页面测试：访问登录、注册、工作台、系统管理、教学管理、校园服务等路由。
- 契约测试：检查接口是否统一返回 `{code,msg,data}`，分页和时间格式是否正确。
- 负向输入测试：发送非法分页、错误 token、疑似 SQL/XSS/路径穿越输入，要求系统受控返回。
- 业务模块测试：检查公告、图书、报修、校园卡、宿舍、课程、考勤、请假、消息等只读接口。
- 接口测试：批量检查公开接口和登录态接口。
- 权限测试：验证匿名、错误 token、学生、教师、管理员访问边界。
- 安全测试：检查敏感路径、错误堆栈、安全响应头。
- 性能测试：多次请求和小并发访问，统计响应耗时和 500 错误。
- 全量测试：执行全部已配置测试，生成完整报告。

## 自动登录

图形验证码不适合自动化识别，因此采用测试环境专用登录旁路。该能力默认关闭，需要被测后端显式配置：

```bash
export CAMPUS_TEST_LOGIN_ENABLED=true
export CAMPUS_TEST_LOGIN_SECRET=campus-test-login
```

测试平台登录时会发送：

```text
X-Test-Login-Secret: campus-test-login
captchaKey=__campus_test_login__
captchaCode=__campus_test_login__
```

只有开关、密钥和固定验证码同时匹配时才会跳过验证码。不开启时仍然走正常验证码登录。
