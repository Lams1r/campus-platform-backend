# 一体化智慧校园系统 - 工作计划安排

## 1. 编制说明
本文档依据《需求分析文档》和《技术规格文档》编制，旨在为“一体化智慧校园系统”的研发、测试与上线提供全面、细致、可执行的项目实施蓝图。本计划明确了项目的总体实施路线、阶段性里程碑、具体的任务拆解、时间进度窗、资源调配及风险管控方案。本文档适用于项目组全体成员（如开发人员、测试人员、实施顾问及项目经理），且符合专业课程设计或毕业设计文档的标准要求，确保项目在预定的8周（约2个月）周期内高质量交付落地。

## 2. 项目实施总体思路
本项目采用**敏捷迭代与瀑布模型相结合**的混合式开发思路：
- **前期统筹（瀑布式）**：集中资源完成需求洞察、系统架构设计与底层数据库规范化搭建，确保基座稳固。
- **中期开发（敏捷迭代）**：遵循“核心优先、通用先行、业务层递进”的原则。采用多Sprint冲刺模式，首选攻克登录鉴权、基础权限等公共底座，随后按业务独立性分批实现课程管理、考勤模块、成绩管理与后勤服务。前后端并行开发，借助统一的接口规范（RESTful JSON）及Mock数据进行解耦联调。
- **后期交付（闭环验工）**：集中进行全链路系统集成测试与压力测试，完成缺陷修复、性能调优部署及操作使用手册的编撰。

## 3. 项目阶段划分
整个项目生命周期严格管控在**8周**内，划分为以下五大核心阶段：
1. **阶段一：项目启动与基建准备阶段（第1周）**
   - 目标：完成需求与技术选型宣贯，搭建基础工程骨架及数据库底层。
2. **阶段二：核心基座与系统管理模块开发阶段（第2-3周）**
   - 目标：打通前后端全链路，完成登录鉴权中心、用户/字典等核心支撑系统的开发与联调。
3. **阶段三：教研核心与学生服务业务开发阶段（第4-6周）**
   - 目标：按领域闭环开发课程管控、考勤网络、成绩流转及宿舍报修、视图面板等业务模块。
4. **阶段四：全链路集成测试与压测联调阶段（第7周）**
   - 目标：前后端全面对接，覆盖所有业务闭环，排查修复系统级Bug，进行防爆破和瞬时高并发压测。
5. **阶段五：部署上线与验收交付阶段（第8周）**
   - 目标：系统正式环境打包部署布网，交付源码、数据库脚本及相关总结手册文档，进行最终结项验收考核。

## 4. 详细工作任务安排
各项工作细化至具体的可执行任务点：

### 4.1 基础环境与基建搭设
*   **前端基建**：初始化 Vite + Vue3 工程模板，整合 Pinia 与 VueRouter；封装 Axios 全局请求与异常拦截器；引入 Element Plus 并调整全局样式变量。
*   **后端基建**：搭建 Spring Boot 核心工程分层（Controller/Service/Mapper）；配置 MyBatis-Plus 及分页插件；集成 Sa-Token 权限配置；挂载 JetCache 缓存组件；集成 Hutool 工具库。
*   **数据库基建**：在 MySQL 8 中建立 `campus_system` 库；基于数据规范执行 DDL 建表脚本（sys_、edu_、campus_ 前缀表）；配置逻辑删除与通用字段（create_time, update_time）自动填充。

### 4.2 第1次冲刺：底层管控与安全基座
*   RBAC表结构设计与入库；实现 Hutool 加密算法工具类封装；开发图形验证码接口。
*   端对端登录登出接口实现及 Sa-Token Token下发与多级鉴权过滤校验逻辑。
*   管理员视角：实现多维度用户台账（支持标准 Excel 模板导出导入）、全局系统字典配置刷新接口。

### 4.3 第2次冲刺：核心教研生态开发
*   **课程模块**：开发课程档案CRUD；教师课件资料安全上传（容量格式限制拦截）；多视图动态课表渲染接口。
*   **考勤模块**：教师端发布限时签到场次（Redis/JetCache暂存）；学生端一键防重签到接口；请假工单提报与审批流转后台接口。

### 4.4 第3次冲刺：成绩流转与各服务面版
*   **成绩模块**：成绩批量模板导入及异常卡段识别；多级成绩提审防篡改流转接口；终端按角色权限拉取成绩/排行视图。
*   **后勤与校园服务**：开发图文并用的宿舍故障报修工单链（状态流转及结单）；校园卡微服务调用/模拟流水数据拉取。
*   **统计大屏与公共服务**：定时任务快照汇聚接口计算全校出勤率及后勤修复率大屏数据；富文本公告精准推发及未读红点状态维护。

## 5. 时间进度安排 (基于8周/56天工作日历)
*   **第1周 (Day 1 - Day 7)**：需求与接口契约锁定，前后端工程初始化，数据库规约建库建表。
*   **第2周 (Day 8 - Day 14)**：开发登录登出、图形验证、Sa-Token 鉴权、用户管理（含Excel导入）、数据字典等公共基础设施。
*   **第3周 (Day 15 - Day 21)**：前后端对接登录及系统管理模块，打通鉴权链路并处理全局拦截异常提示反馈。
*   **第4周 (Day 22 - Day 28)**：进入高密度业务，并行开发“课程管控模块”与“动态考勤及请假审批网络”。
*   **第5周 (Day 29 - Day 35)**：业务推进，火力集中开发“多重成绩流转核验防改模块”与“富文本公告通知机制”。
*   **第6周 (Day 36 - Day 42)**：收尾业务板块，开发“宿舍报修及状态流转”、“校园卡消费流水查验”及复杂聚合的“全视界数据宏观大屏”。
*   **第7周 (Day 43 - Day 49)**：全面进入集成联调期。全流程业务闭环穿透测试，修复逻辑漏洞；实施突发高并发查分/签到模拟压测并优化慢SQL及JetCache策略。
*   **第8周 (Day 50 - Day 56)**：代码合规性审查及清理，Fat-Jar 后端包及 Vite 前端静态页产出打包。部署至服务器环境（如Nginx + JDK17），编写《系统操作手册》，准备并进行结项路演验收考核。

## 6. 各阶段交付成果
1. **基建阶段**：MySQL数据库结构全集脚本文件（.sql）、基于 Postman/Apifox 导出的可执行 API 接口契约说明书。
2. **开发阶段（各Sprint结束）**：可独立运行的业务模块前后端代码分支、单元自测通过的测试报告简报。
3. **测试阶段**：系统集成测试缺陷清单及Bug清零修复记录报表、高压测试并发能耗报告（如 JMeter 输出）。
4. **最终交付**：
   * 完整且经过安全脱敏的系统工程原码及打包部署应用载体。
   * 《安装部署说明文档》与规范化的《用户操作指南手册》（图文并茂）。
   * 结题答辩或总结演示汇报用的 PPT 及相关材料。

## 7. 项目重点与难点安排
*   **难点一：高并发签到考勤的瞬间洪峰处理**。解决策略：利用 JetCache 实施预检拦截机制，剥离直接强打数据库的短促高峰流量；通过（场次ID+用户ID）缓存锁构建防重复提交卡口。安排在第4周与第7周调优测试中重点照顾。
*   **难点二：成绩流转的强状态一致性约束**。解决策略：后端Service严控状态机流转；一经变动瞬间废止接口写入通道。数据库层不可有物理外键，纯依靠 Java 业务断点抛出违规异常。在第5周实现时需编写充分的分支测试代码覆盖验证。
*   **难点三：巨型动态大屏的数据读取拖累**。解决策略：坚决执行后台定时任务（如 Spring Task）轮询计算大盘指标，生成快照并推送至独立的高速缓存池中供前端大屏实时拉取提取，绝对切断大屏实时触发联合全表 COUNT/SUM 这种拖死系统的动作方案。

## 8. 风险控制与保障措施
*   **进度延期风险**：严格推行敏捷冲刺站会制度。当功能出现“死磕”卡壳超过一日，立即降级处理或上报求援；优先保障教学主干链路，非核心增强功能（如复杂的邮件发送、外部实态刷卡直连等）可暂时以挡板（Mock）数据平替保证流程演示打通。
*   **数据损毁风险**：所有的数据库增删改查底层严格服从逻辑删除（`is_deleted`）安全底线原则；重要大表更新需绑定强硬事务（@Transactional），保障业务数据如遇宕机也能实现安全同卷回退。
*   **技术适配风险**：针对 Vue3 及 Vite 组合的新手磨合阵痛期，项目启动首周将直接锁定核心包与底层环境版本（如强绑定 JDK17+MySQL8.0.32），防范包依赖版本跨越更新突变引发的群发系统粉碎性崩溃。

## 9. 工作团队及职责划分
鉴于本项目主要服务于课程设计/毕业设计或中小型科研孵化规模，职责按微型敏捷团队结构设置：
*   **项目经理兼系统架构师 (1人)**：负责统筹前中后三期总进度节拍、把控数据库基底建制与底层拦截器、全局鉴权（Sa-Token）框架安全门搭建与整体系统终极联调发布。（核心底座把控者）
*   **后端业务开发工程师 (1-2人)**：承担并拆分攻坚系统模块中复杂业务逻辑（如考勤流、成绩流转、高强度导出导入运算处理），并对下负责独立业务表的数据维护接口和业务代码。
*   **前端交互与界面工程师 (1-2人)**：负责搭建基于 Element Plus 的各类展示板与控制组件，根据 API 蓝图衔接调取后方接口及实现动态路由鉴权及页面全局状态树（Pinia）的管理和界面还原与调整。
*   **测试与实施专员 (兼管或1人)**：编写测试案例脚本，把关质量红线的边缘测试（格式拦截、容量限制压测、模拟越界探测），并担纲项目落成后的用户指导手册撰写及现场服务器的搭设搬迁上线工作部署。
